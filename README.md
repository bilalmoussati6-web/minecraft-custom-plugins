# SoulItem

Custom Minecraft item plugin generated by **Custom Item Maker**.

## This is a REAL Custom Item Plugin (NOT a texture pack)

This plugin registers **`soul_blade`** as a real server-side custom item
via Bukkit's **PersistentDataContainer** (NBT tag system). It is NOT simply
a texture pack that overrides a vanilla item.

### What this means

| Without the plugin | With the plugin |
|--------------------|----------------|
| A regular `DIAMOND_SWORD` — nothing special | Your custom `soul_blade` with all abilities, attributes, and recipes |
| Any `DIAMOND_SWORD` in the world is just vanilla | Only `DIAMOND_SWORD`s stamped with the plugin's NBT tag become `soul_blade` |
| No abilities fire | All abilities, crafting, and the give command work |

The item's true identity is its **NBT tag**, NOT its CustomModelData or display name.
The **resource pack is fully optional** — it only changes the visual texture.
Without it, the item still works, it just looks like `DIAMOND_SWORD`.

## Build

```bash
mvn clean package
```

Drop the generated JAR from `target/` into your server's `plugins/` folder.

## In-Game Commands

- `/givesoulitem` — gives yourself the **real** `soul_blade` custom item (NBT-stamped by the plugin)
- `/givesoulitem <player>` — gives the item to another player

The item given by these commands is a **real plugin item** — it has the NBT tag
that lets the plugin recognize it and fire all abilities. Without the NBT tag,
a `DIAMOND_SWORD` is just a regular vanilla item.

## Resource Pack (OPTIONAL)

A starter resource pack is in the `resourcepack/` folder. **This is fully optional.**

1. Zip the contents of `resourcepack/` (NOT the folder itself).
2. Hand it to your players (or host it and set `resource-pack:` in `server.properties`).
3. The custom texture is mapped to `custom_model_data: 1001` on `DIAMOND_SWORD`.
4. **Without the resource pack**: the item still works, but looks like `DIAMOND_SWORD`.

## Custom Head Texture (Optional)


This plugin does NOT have a custom head texture configured. The item will look
like `DIAMOND_SWORD` unless a resource pack is used.

**To add a custom head texture (no resource pack needed!):**

1. Design your item texture in the painter
2. Download the PNG from the painter's download button
3. Upload it to **[mineskin.org](https://mineskin.org/create)** — you'll get a "Texture Value" (base64)
4. OR browse pre-made heads at **[minecraft-heads.com](https://minecraft-heads.com/custom-heads)**
5. Copy the base64 texture value and paste it in the "Custom Head Texture" field in the Identity tab
6. Regenerate the plugin ZIP and reinstall
7. The item will now be a unique player head with the custom texture!
